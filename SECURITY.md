# Security Policy

## Supported Versions

- `main` / `master`
- Последний production release, если он опубликован отдельно

## Reporting

- Если в репозитории включён GitHub Private Vulnerability Reporting, используйте его для любых уязвимостей, которые не должны раскрываться публично.
- Если приватный канал недоступен, не публикуйте exploit и детали атаки в открытом issue. Создайте минимальный issue без деталей и запросите безопасный канал связи.
- Для низкорисковых проблем качества данных, конфигурации или документации можно открывать обычные issues.

## Scope

- Уязвимости зависимостей `npm` и `go`
- Ошибки аутентификации, сессий, cookie и rate limiting
- Проблемы SSRF/XSS/IDOR/CSRF/privilege escalation
- Утечки секретов, приватных данных и production-конфигурации
